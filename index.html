Multiple Inheritance dengan Mixin, Polimorfisme, dan Abstraksi

Dalam pemrograman berorientasi objek, terdapat beberapa konsep penting seperti multiple inheritance dengan mixin, polimorfisme, dan abstraksi yang saling berkaitan.

Multiple Inheritance dengan Mixin digunakan untuk menambahkan beberapa fitur ke dalam satu class tanpa menggunakan pewarisan langsung. Dengan mixin, sebuah class dapat memiliki lebih dari satu kemampuan.

Polimorfisme memungkinkan satu tipe data digunakan untuk berbagai objek dengan perilaku yang berbeda. Hal ini membuat kode lebih fleksibel dalam penggunaan.

Abstraksi digunakan untuk menyederhanakan sistem dengan cara menyembunyikan detail yang tidak diperlukan dan hanya menampilkan fungsi utama kepada pengguna.

Ketiga konsep ini membantu dalam membuat program yang lebih terstruktur, mudah dikembangkan, dan mudah dipelihara.

Studi Kasus Burung Dan Bebek

Burung merupakan class induk, sedangkan Bebek adalah class turunan. Bebek mewarisi sifat dari Burung, tetapi dapat memiliki perilaku yang berbeda.

Hal ini menunjukkan konsep inheritance dan polimorfisme dalam OOP.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

mixin CanFly {
  void fly() {
    print('Can fly!');
  }
}

mixin CanSwim {
  void swim() {
    print('Can swim!');
  }
}

class Bird with CanFly {}
class Duck extends Bird with CanSwim {}

void main() {
  print("=== MIXIN BASIC ===");

  Bird bird = Bird();
  bird.fly();

  Duck duck = Duck();
  duck.fly();
  duck.swim();
}

STUDI KASUS MIXIN DENGAN STATE

Mixin dengan state digunakan untuk menambahkan fitur sekaligus menyimpan data (state) ke dalam sebuah class tanpa harus menggunakan pewarisan langsung.

Dengan mixin, sebuah class dapat memiliki beberapa kemampuan tambahan beserta data yang mendukung fungsinya.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

mixin CanFly {
  bool _isFlying = false;

  void fly() {
    if (!_isFlying) {
      _isFlying = true;
      print('Started flying!');
    }
  }

  void land() {
    if (_isFlying) {
      _isFlying = false;
      print('Landed!');
    }
  }
}

mixin CanSwim {
  bool _isSwimming = false;

  void swim() {
    if (!_isSwimming) {
      _isSwimming = true;
      print('Started swimming!');
    }
  }

  void stopSwim() {
    if (_isSwimming) {
      _isSwimming = false;
      print('Stop swimming!');
    }
  }
}

class Bird with CanFly {}
class Duck extends Bird with CanSwim {}

void main() {
  print("=== MIXIN STATE ===");

  Bird bird = Bird();
  bird.fly();
  bird.land();

  Duck duck = Duck();
  duck.fly();
  duck.swim();
}

Pengantar Class Modifiers

Class modifiers adalah kata kunci dalam Dart yang digunakan untuk mengatur bagaimana sebuah class dapat digunakan, diwariskan, atau dibatasi aksesnya. Dengan adanya modifiers, developer dapat mengontrol perilaku class agar sesuai dengan kebutuhan program.

Setiap modifier memiliki fungsi yang berbeda, seperti membatasi pewarisan, mencegah perubahan class, atau mengatur implementasi class turunan. Hal ini membantu menjaga struktur program tetap aman dan terorganisir.

Kenapa menggunakan class modifiers:

  • Mengontrol bagaimana class digunakan dan diwariskan
  • Mencegah penggunaan class secara sembarangan
  • Menjaga konsistensi dan keamanan struktur program
  • Mempermudah pengelolaan kode dalam skala besar

Sealed Class

Sealed class adalah class yang membatasi pewarisan hanya dalam satu file. Class ini digunakan untuk mengontrol subclass agar tidak dibuat sembarangan dan memastikan semua kemungkinan kondisi dalam program dapat ditangani dengan jelas.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);

  @override
  String toString() {
    return "Success: $message";
  }
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);

  @override
  String toString() {
    return "Error: $errorMessage";
  }
}

void main() {
  print("=== SEALED CLASS ===");

  ApiResponse res1 = Success("Selamat datang Uliya");
  ApiResponse res2 = Error("Password salah");

  print(res1);
  print(res2);
}

Keuntungan Sealed Class

  • Membatasi pembuatan subclass hanya dalam satu file sehingga lebih terkontrol
  • Meningkatkan keamanan karena tidak semua class bisa mewarisi
  • Mempermudah penanganan kondisi dengan pattern matching
  • Mencegah kemungkinan kondisi yang tidak tertangani dalam program
DART SOURCE CODE
// Nama   : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

void handleResponse(ApiResponse response) {
  if (response is Success) {
    print("Success: ${response.message}");
  } else if (response is Error) {
    print("Error: ${response.errorMessage}");
  }
}

void main() {
  print("=== KEUNTUNGAN SEALED CLASS ===");

  ApiResponse res1 = Success("Selamat datang Uliya");
  ApiResponse res2 = Error("Password salah");

  handleResponse(res1);
  handleResponse(res2);
}

Base Class

Base class adalah class yang hanya bisa diwariskan menggunakan extends dan tidak dapat diimplementasikan secara bebas di luar library. Class ini digunakan untuk mengontrol pewarisan agar tetap terstruktur.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

base class Animal {
  void sound() {
    print("Hewan mengeluarkan suara");
  }
}

base class Lion extends Animal {
  @override
  void sound() {
    print("Singa mengaum");
  }
}

void main() {
  print("=== BASE CLASS ANIMAL ===");

  Animal animal = Lion();
  animal.sound();
}

Kapan Menggunakan Base Class

Base class digunakan ketika ingin membatasi pewarisan class agar hanya dapat dilakukan dengan extends dan tetap berada dalam kontrol yang jelas. Dengan base class, pengembang dapat menjaga struktur program agar tidak digunakan secara sembarangan.

Penggunaan base class membantu memastikan bahwa class turunan mengikuti aturan yang sudah ditentukan, sehingga program menjadi lebih terstruktur dan konsisten.

Kapan digunakan:

  • Saat ingin membatasi pewarisan hanya dengan extends
  • Saat ingin menjaga struktur class tetap konsisten
  • Saat ingin mencegah penggunaan implements yang tidak sesuai
  • Saat membutuhkan kontrol penuh terhadap class turunan
DART SOURCE CODE
// Nama   : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

base class Animal {
  void move() {
    print("Hewan bergerak");
  }

base class Elephant extends Animal {

  }

void main() {

Final Class

Final class adalah class yang tidak dapat diwariskan atau di-extend oleh class lain. Final class digunakan ketika sebuah class tidak ingin diwariskan agar implementasi tetap aman dan tidak diubah oleh class lain.

Kegunaan:

  • Mencegah class digunakan sebagai parent
  • Menjaga implementasi agar tidak diubah oleh subclass
  • Meningkatkan keamanan dan kestabilan kode
DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

final class Settings {
  final String theme;
  final int fontSize;

  Settings({required this.theme, required this.fontSize});

  void showSettings() {
    print("Theme: $theme, Font Size: $fontSize");
  }
}

void main() {
  print("=== FINAL CLASS ===");

  Settings settings = Settings(theme: "Dark", fontSize: 14);
  settings.showSettings();
}

. Perbandingan Class Modifers

Modifier Deskripsi
abstract Tidak bisa dibuat objek langsung, digunakan sebagai dasar class
base Hanya dapat diwariskan dalam library yang sama
interface Digunakan sebagai kontrak yang harus diimplementasikan
final Tidak dapat diwariskan atau di-extend
sealed Membatasi subclass hanya dalam file yang sama

Praktik 1 Sealed Class dengan Pattern Matching

Sealed class memungkinkan kita mengetahui semua kemungkinan subclass yang ada dalam satu file.

Dengan pattern matching, setiap kemungkinan tersebut dapat diperiksa dan ditangani secara spesifik sesuai jenis objeknya.

Hal ini membuat kode lebih aman karena semua kondisi dapat ditangani tanpa ada yang terlewat.

DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

sealed class Status {}

class Loading extends Status {}

class Loaded extends Status {
  final String data;
  Loaded(this.data);
}

class Failed extends Status {
  final String error;
  Failed(this.error);
}

void handleStatus(Status status) {
  switch (status) {
    case Loading():
      print('Loading...');
    case Loaded(:final data):
      print('Data: $data');
    case Failed(:final error):
      print('Error: $error');
  }
}

void main() {
  print("=== SEALED PATTERN ===");

  handleStatus(Loading());
  handleStatus(Loaded("Data berhasil"));
  handleStatus(Failed("Terjadi kesalahan"));
}

Sistem State Management

State management adalah cara mengatur kondisi aplikasi seperti loading, success, dan error. Dengan sealed class, setiap state dapat didefinisikan dengan jelas.

Fitur:

  • Mengelola kondisi aplikasi
  • Mengatur alur program

Kenapa digunakan:

  • Agar kondisi aplikasi lebih terstruktur
  • Mencegah error yang tidak ter-handle

Batasan:

  • Bisa menjadi kompleks jika state terlalu banyak
DART SOURCE CODE
// Nama   : Uliya Gumpari
// Nim    : 251410008
// Kelas  : SI2A

sealed class AppState {}

class LoadingState extends AppState {}

class SuccessState extends AppState {
  final List items;
  SuccessState(this.items);
}

class ErrorState extends AppState {
  final String message;
  ErrorState(this.message);
}

void updateUI(AppState state) {
  switch (state) {
    case LoadingState():
      print('Loading...');
    case SuccessState(:final items):
      print('Success: $items');
    case ErrorState(:final message):
      print('Error: $message');
  }
}

void main() {
  print("=== APP STATE ===");

  updateUI(LoadingState());
  updateUI(SuccessState(["A", "B"]));
  updateUI(ErrorState("Gagal"));
}

Ringkasan & Best Practices

Ringkasan:

  • Sealed class digunakan untuk membatasi hierarki dan mendukung pattern matching
  • Base class digunakan sebagai dasar class yang hanya dapat di-extend
  • Final class digunakan untuk mencegah pewarisan dari luar

Best Practices:

  • Gunakan sealed class untuk state atau kondisi yang jumlahnya terbatas
  • Gunakan base class untuk membuat kerangka dasar program
  • Gunakan final class untuk class yang tidak ingin diubah atau diwariskan

Bonus Base Class Repository

Repository menggunakan base class agar hanya bisa diwariskan menggunakan extends, bukan diimplementasikan sembarangan.

Penggunaan base atau abstract class bertujuan untuk mencegah implementasi langsung yang tidak sesuai, sehingga hanya class turunan yang mengikuti aturan yang dapat digunakan.

Fitur Base Class pada Repository:

  • Hanya bisa diwariskan dengan extends
  • Tidak bisa digunakan dengan implements di luar library
  • Menjaga struktur method tetap sama

Kenapa menggunakan base:

  • Menjaga struktur CRUD tetap konsisten
  • Mencegah implementasi yang tidak sesuai
  • Membatasi penggunaan class agar lebih terkontrol

Batasan:

  • Tidak bisa diimplementasikan menggunakan implements
DART SOURCE CODE
// Nama   : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

base class Repository {
  void create(String data) {
    print("Create: $data");
  }

  void read() {
    print("Read data");
  }

  void update(int id, String data) {
    print("Update id=$id: $data");
  }

  void delete(int id) {
    print("Delete id=$id");
  }
}

base class UserRepository extends Repository {
  @override
  void read() {
    print("Menampilkan semua user");
  }
}

void main() {
  print("=== REPOSITORY USER ===");

  UserRepository userRepo = UserRepository();
  userRepo.create("Uliya");
  userRepo.read();
  userRepo.update(1, "Uliya Gumpari");
  userRepo.delete(1);
}

Bonus Final Class MathUtils

MathUtils menggunakan final class karena hanya berisi fungsi utilitas dan tidak perlu diwariskan ke class lain.

Penggunaan final class bertujuan untuk memastikan bahwa class ini tidak dapat diwariskan sehingga fungsi di dalamnya tetap aman dan tidak diubah oleh class lain.

Fitur Final Class pada MathUtils:

  • Tidak bisa di-extend oleh class lain
  • Tidak bisa diimplementasikan
  • Cocok untuk method static

Kenapa menggunakan final:

  • Mencegah class diwariskan
  • Menjaga fungsi tetap sederhana dan aman
  • Menghindari perubahan dari subclass

Batasan:

  • Tidak bisa diwariskan sama sekali
DART SOURCE CODE
// Nama   : Uliya Gumpari
// NIM   : 251410008
// Kelas : SI2A

final class MathUtils {
  static int tambah(int a, int b) => a + b;
  static int kurang(int a, int b) => a - b;
  static int kali(int a, int b) => a * b;
  static double bagi(double a, double b) => a / b;
}

void main() {
  print("=== MATH UTILS ===");

  print("Tambah: ${MathUtils.tambah(10, 5)}");
  print("Kurang: ${MathUtils.kurang(10, 5)}");
  print("Kali: ${MathUtils.kali(10, 5)}");
  print("Bagi: ${MathUtils.bagi(10, 5)}");
}
